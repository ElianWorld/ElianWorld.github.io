Android arouter路由实记

2021-01-13

1、不同组件的路由group必须不同

arouter在编译期间对工程中的不同组件的路由关系进行校验，不同组件之间、组件和主工程之间的group必须不同

2、所有有需要路由跳转的对象，需要在主工程进行注册

arouter不支持插件化的热加载，需要在主工程dependencies手动注册对应的组件后，才可以实现路由跳转

3、最后在统一的资源组件中对路由关系进行维护

可以选择在资源组件中对工程中不同组件的路由关系进行手动维护，优势是可以添加路由的注释，便于后期维护，也可以把路由和组件间的对应关系也描述在资源组件中

4、新添加路由后，最好卸载重新运行app

实际运行期间，如果不卸载重新运行，可能找不到路由报错

5、组件资源的命名最好以group开头，防止因为资源冲突导致编译错误

APK在打包过程中会对资源进行统一的打包，有兴趣的可以看下APK打包流程和AAPT的实现逻辑，在实际使用过程中，为防止资源的冲突，最好将资源的命名以group为开头，保证编译不出问题

6、如果代码编写过程中出现资源加载异常，比如通过R定位不到对应的资源了，最好Invalidate Caches / Restart

7、使用arouter的组件中，gradle文件的defaultConfig中务必添加以下设置

        javaCompileOptions {
            annotationProcessorOptions {
                arguments = [moduleName: project.getName()]
            }
        }
